## Design Context

### Users
開發者（包含作者本人及其他開發者），日常開發場景使用。主要用途：管理多個專案、操作終端機、監控開發伺服器、追蹤 Git 狀態與 Claude API 用量。使用者熟悉 IDE 與終端機操作，期望工具不擋路、快速上手。

### Brand Personality
簡潔、簡單、好用。Kujira 是一個安靜的工具 — 不搶眼、不花俏，在需要時提供資訊，不需要時退到背景。

### Aesthetic Direction
- **視覺基調**: macOS 原生工具感，IDE 風格的資訊密度。暗色主題為主（6 款主題全為暗色）。
- **設計語言**: 扁平、低對比邊框、微妙的層次（bg-primary → secondary → tertiary → elevated）。圓角 6-8px。hover 狀態用極淡的白色覆蓋層（rgba 255,255,255, 0.04-0.07）。
- **字體**: macOS 系統字體（-apple-system, SF Pro Text），monospace 用於程式碼與終端。不使用裝飾性字體。
- **色彩**: CSS 變數系統，語意化命名。accent 色用於狀態指示（green=成功/執行、yellow=警告/修改、red=錯誤/刪除、blue=資訊/連結、purple=Git 分支）。中性色帶冷灰色調。
- **動態**: 極簡 — 僅 hover transition（0.15s）與面板寬度過渡（0.2s ease）。無彈跳、無華麗動畫。

### Design Principles
1. **工具透明性** — UI 是達成目的的手段，不是目的本身。每個元素都必須有明確功能。
2. **資訊密度優先** — 在有限空間內呈現最多有用資訊，但不擁擠。善用字體大小層次（10-14px）和色彩區分。
3. **一致的視覺語言** — 所有元件遵循同一套 CSS 變數、間距規則、圓角尺寸。新元件必須融入現有體系，不引入新的設計模式。
4. **狀態即時可見** — 伺服器狀態、Git 變更、Claude 用量 — 重要狀態用色彩編碼一眼可辨。
5. **macOS 原生感** — 尊重平台慣例（交通燈按鈕、系統字體、細緻 scrollbar），讓使用者感覺這是一個屬於 macOS 的 app。
